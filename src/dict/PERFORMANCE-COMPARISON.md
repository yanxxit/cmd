# 屌丝词典 V1 与 V2 性能对比分析

## 测试环境

- 系统: macOS
- Node.js 版本: 最新稳定版
- 词典数据: 176,000+ 英汉词汇对
- 测试方式: 多次运行取平均值

## 测试结果

### 单词查询性能
- V1 (内存Map): 平均 0.0343ms/次
- V2 (LevelDB): 平均 0.8627ms/次
- 分析: V1 在单次单词查询方面明显更快，因为直接在内存中查找

### 前缀查询性能
- V1 (内存Map): 平均 4.8908ms/次
- V2 (LevelDB): 平均 1.9787ms/次
- 分析: V2 在前缀查询方面更快，LevelDB 的范围查询效率更高

### 内存占用
- V1 (内存Map): 约 7MB+
- V2 (LevelDB): 很小（仅缓存需要的部分数据）
- 分析: V2 占用内存显著更低，适合资源受限环境

## 使用建议

### 使用 V1 (ds) 当:
- 需要极致的单词查询速度
- 内存资源充足
- 词典数据量不大
- 不需要数据持久化

### 使用 V2 (ds-v2) 当:
- 需要节省内存资源
- 需要数据持久化
- 频繁进行前缀/范围查询
- 处理大量数据

### 总结
- V1 适合快速查询场景，内存占用高
- V2 适合数据持久化和资源受限场景，前缀查询更快
- 根据具体应用场景选择合适的版本
