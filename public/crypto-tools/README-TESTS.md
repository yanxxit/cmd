## 🧪 测试用例详解

### 基础功能测试

#### 1. 正常加密解密流程
**目的**: 验证基本的加密和解密功能是否正常工作

**测试步骤**:
1. 生成随机密钥（32 位）和偏移量（16 位）
2. 加密文本 "Hello, World!"
3. 使用相同密钥和偏移量解密
4. 验证解密结果与原文一致

**预期结果**: ✅ 解密结果与原文完全一致

#### 2. 空字符串加密
**目的**: 测试空字符串的加密和解密

**测试步骤**:
1. 加密空字符串 ""
2. 解密加密结果
3. 验证解密结果为空字符串

**预期结果**: ✅ 空字符串可以正常加密和解密

#### 3. 单字符加密
**目的**: 测试单个字符的加密和解密

**测试步骤**:
1. 加密单字符 "A"
2. 解密加密结果
3. 验证解密结果为 "A"

**预期结果**: ✅ 单字符可以正常加密和解密

### 边界情况测试

#### 4. 超长文本加密
**目的**: 测试 10000 字符的超长文本加密解密

**测试步骤**:
1. 生成 10000 个字符的文本（"A"重复）
2. 加密超长文本
3. 解密加密结果
4. 验证解密结果长度和内容

**预期结果**: ✅ 超长文本可以正常加密和解密

#### 5. 特殊字符 - 换行符
**目的**: 测试包含换行符的文本

**测试步骤**:
1. 加密包含换行符的文本 "Line 1\nLine 2\nLine 3"
2. 解密加密结果
3. 验证换行符保留

**预期结果**: ✅ 换行符在加密解密后保持不变

#### 6. 特殊字符 - 制表符
**目的**: 测试包含制表符的文本

**测试步骤**:
1. 加密包含制表符的文本 "Col1\tCol2\tCol3"
2. 解密加密结果
3. 验证制表符保留

**预期结果**: ✅ 制表符在加密解密后保持不变

#### 7. 特殊字符 - 所有 ASCII 字符
**目的**: 测试所有可打印 ASCII 字符

**测试步骤**:
1. 加密所有可打印 ASCII 字符
2. 解密加密结果
3. 验证所有字符都正确还原

**预期结果**: ✅ 所有 ASCII 字符都可以正确加密和解密

## 🔧 加密参数

所有测试使用以下标准参数：
- **加密算法**: AES
- **加密模式**: CBC (Cipher Block Chaining)
- **填充方式**: PKCS7
- **密钥长度**: 128/192/256 位（16/24/32 字符）
- **偏移量长度**: 128 位（16 字符）
- **编码方式**: Base64
