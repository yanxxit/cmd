# UUID v7 时间戳验证报告

## 📊 测试概述

本测试报告验证 UUID v7 生成算法中时间戳提取的准确性，确保从 UUID v7 解析出的时间戳与输入时间完全一致。

---

## 🎯 测试方法

### 测试流程
```
1. 生成 UUID v7（使用已知时间戳）
   ↓
2. 从 UUID v7 提取前 48 位时间戳
   ↓
3. 将提取的时间戳与原始输入对比
   ↓
4. 计算误差（应 ≤ 1ms）
```

### 验证公式
```javascript
// UUID v7 结构
UUID v7: xxxxxxxx-xxxx-7xxx-xxxx-xxxxxxxxxxxx
         └─ 48 位时间戳 ─┘

// 提取时间戳
timestamp_hex = uuid[0:12]  // 前 12 个十六进制字符
timestamp = parseInt(timestamp_hex, 16)

// 验证
error = |timestamp - original_time|
pass if error ≤ 1ms
```

### 十六进制转换验证

```javascript
// 验证函数
function verifyTimestamp(uuid, originalTime) {
  const hex = uuid.replace(/-/g, '').slice(0, 12);
  const extracted = parseInt(hex, 16);
  const error = Math.abs(extracted - originalTime);
  
  console.log(`UUID: ${uuid}`);
  console.log(`时间戳高位 (hex): ${hex.slice(0, 6)}`);
  console.log(`时间戳低位 (hex): ${hex.slice(6)}`);
  console.log(`提取时间戳：${extracted} ms`);
  console.log(`原始时间戳：${originalTime} ms`);
  console.log(`误差：${error} ms`);
  console.log(`状态：${error <= 1 ? '✅' : '❌'}`);
}

// 示例
verifyTimestamp('01902f4a-8c1a-7abc-def0-123456789abc', 1745419230123);
```

## 🎯 使用建议

### 何时使用 UUID v7

✅ **推荐场景**：
- 需要按时间排序的唯一 ID
- 分布式系统主键生成
- 数据库索引优化（时间局部性）
- 日志追踪（可提取时间）

❌ **不推荐场景**：
- 需要完全随机的 ID（使用 UUID v4）
- 需要隐藏时间信息（使用 UUID v4）
- 需要人类可读的短 ID（使用 NanoID）
