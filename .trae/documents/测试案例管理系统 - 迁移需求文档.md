# 测试案例管理系统 - 迁移需求文档

## 项目概述

### 项目名称
Admin System - 后台管理系统（包含测试案例管理模块）

### 项目描述
一个基于 Web 的综合后台管理系统，首先实现“测试案例管理”模块。用于管理 API 测试案例，支持案例的增删改查、分组、搜索、标签过滤和统计分析。

### 技术栈（目标）
- **前端框架**: Next.js 14+ (Pages Router，根据当前项目结构调整)
- **UI 组件库**: Ant Design 5.x
- **编程语言**: TypeScript
- **状态管理**: ahooks (useRequest, useUrlState 等，当前项目已安装)
- **数据验证**: Ant Design Form 自带验证
- **HTTP 客户端**: fetch / ahooks useRequest
- **样式方案**: Tailwind CSS
- **图标库**: @ant-design/icons
- **编辑器**: @uiw/react-codemirror (当前项目已安装，用于 JSON 编辑)

## 功能需求

### 1. 后台管理系统 Shell (Admin Layout)

**功能描述**: 提供后台系统通用的骨架，包括侧边栏导航、顶部信息栏和内容区域。

**需求详情**:
1. **侧边栏 (Sidebar)**
   - 左侧固定导航栏
   - 包含菜单：仪表盘 (Dashboard)、测试案例管理 (Test Cases)
   - 支持折叠/展开
   - 深色/浅色主题支持 (跟随 Ant Design ConfigProvider)

2. **顶部导航 (Header)**
   - 面包屑导航 (Breadcrumb)
   - 用户信息展示 (Avatar, Username)
   - 退出登录等操作

3. **内容区域 (Content)**
   - 自适应高度
   - 统一的白色背景和圆角卡片包裹
   - 全局加载状态 (Spin) 支持

---

### 2. 数据模型 (测试案例)

#### 2.1 测试案例数据结构

```typescript
export interface TestCase {
  _id?: string;              // 唯一标识（自动生成）
  apiName: string;           // 接口名称，如：/api/users
  title: string;             // 案例标题，如：查询用户列表 - 正常场景
  requestParams?: object;    // 请求参数（JSON 对象）
  responseData?: object;     // 返回数据（JSON 对象）
  remark?: string;           // 备注说明
  tags: string[];            // 标签数组
  requestTime?: string;      // 请求时间（ISO 8601）
  createdAt?: string;        // 创建时间（自动生成）
  updatedAt?: string;        // 更新时间（自动更新）
}
```

### 3. 功能模块

#### 3.1 案例列表管理

**功能描述**: 展示测试案例列表，支持分页、搜索、过滤

**需求详情**:
1. **列表展示**
   - 使用 Ant Design `Table` 组件展示案例
   - 显示字段：接口名、标题、标签、请求时间、创建时间、操作
   - 支持按创建时间倒序/正序排列
   - 默认每页 20 条，使用前端分页或服务端分页（对接现有 API）

2. **搜索功能**
   - 关键词搜索：支持接口名、标题模糊匹配
   - 接口名分组筛选：下拉选择已存在的接口名
   - 标签过滤：下拉选择已存在的标签

3. **批量操作**
   - 支持多选案例
   - 批量删除功能

4. **交互要求**
   - 点击行查看详情（或者操作栏“详情”按钮）
   - 加载状态提示

**API 接口**: `GET /api/test-cases`

---

#### 3.2 案例详情查看

**功能描述**: 查看单个测试案例的完整信息

**需求详情**:
1. **弹窗展示**
   - 使用 Ant Design `Modal` 模态框展示详情

2. **信息展示**
   - 基本信息：接口名、案例标题
   - 请求参数：JSON 格式化展示，支持代码高亮（如 react-codemirror）
   - 返回数据：JSON 格式化展示
   - 备注和标签：卡片式展示
   - 提供"编辑"按钮，点击切换到编辑弹窗

---

#### 3.3 新建/编辑案例

**功能描述**: 创建新案例或编辑现有案例

**需求详情**:
1. **表单布局**
   - 使用 Ant Design `Form` 组件
   - 接口名称（必填）
   - 案例标题（必填）
   - 请求参数：JSON 编辑器（或 TextArea 支持 JSON 验证）
   - 返回数据：JSON 编辑器（或 TextArea 支持 JSON 验证）
   - 备注：多行文本框
   - 标签：`Select` (mode="tags") 组件展示与输入
   - 请求时间：`DatePicker` (showTime)

2. **交互要求**
   - 保存时显示 loading 状态
   - 成功/失败提示 (Message)
   - 保存后自动刷新列表

**API 接口**:
- `POST /api/test-cases` (新建)
- `PUT /api/test-cases/:id` (更新)

---

#### 3.4 删除案例

**功能描述**: 删除单个或多个案例

**需求详情**:
1. **单个删除**: 操作栏删除按钮，使用 `Popconfirm` 二次确认
2. **批量删除**: 列表多选后触发，使用 `Modal.confirm` 二次确认

**API 接口**:
- `DELETE /api/test-cases/:id`
- `POST /api/test-cases/batch` (operation: 'delete')

### 5. 联调要求

后端 API 地址为 `http://localhost:3000/api/test-cases`（由于 next-app 在 3030，可能后端在另一个端口，需配置 API 请求的 baseURL。或者根据实际后端地址进行调整。这里暂定 API 前缀需可配，开发时可通过代理或绝对 URL 请求）。

### 6. 开发步骤

1. **基础布局搭建**: 实现 `AdminLayout`，集成 Ant Design `Layout`、`Menu`。
2. **页面路由创建**: 在 `pages/admin/test-cases/index.tsx` 引入 `AdminLayout`。
3. **数据请求封装**: 使用 `ahooks` 的 `useRequest` 封装测试案例的 CRUD 请求。
4. **列表与筛选实现**: 开发 `TestCaseTable` 和 `TestCaseFilters` 组件。
5. **弹窗与表单实现**: 开发 `TestCaseModal` 和 `TestCaseDetail`。
6. **JSON 编辑器集成**: 引入 `@uiw/react-codemirror` 和 `@codemirror/lang-json`（或者简单使用 TextArea + JSON.parse 校验）。
7. **全链路测试**: 测试增删改查、分页、搜索等功能。
