# 测试案例记录系统 - 实施计划

## 项目概述
基于 @yanit/jsondb 实现一个完整的测试案例记录管理系统，支持接口测试案例的增删改查、分组展示、搜索过滤等功能。

---

## 📋 实施步骤

### 1. 创建 Model 层 (src/model/jsondb/TestCase.js)
- 创建 `TestCaseModel` 类，参考 `TaskManagerModel` 结构
- 设计测试案例数据结构：
  - `_id`: 唯一标识
  - `apiName`: 接口名称
  - `title`: 案例标题
  - `requestParams`: 请求参数 (JSON)
  - `responseData`: 返回数据 (JSON)
  - `remark`: 备注
  - `tags`: 标签数组
  - `requestTime`: 请求时间戳
  - `createdAt`: 创建时间
  - `updatedAt`: 更新时间
- 实现基础 CRUD 方法：
  - `create(data)`: 创建案例
  - `findById(id)`: 查找单个案例
  - `find(options)`: 查询案例列表（支持分页、搜索、标签过滤）
  - `update(id, data)`: 更新案例
  - `delete(id)`: 删除案例
  - `batchDelete(ids)`: 批量删除
  - `getStats()`: 获取统计信息
  - `getGroupedApiNames()`: 获取分组接口名列表

### 2. 创建 API 路由层 (src/http-server/test-case-api.js)
- 实现 RESTful API 接口：
  - `GET /api/test-cases`: 查询案例列表（支持搜索、分页、标签过滤）
  - `GET /api/test-cases/:id`: 获取单个案例详情
  - `POST /api/test-cases`: 创建新案例
  - `PUT /api/test-cases/:id`: 更新案例
  - `DELETE /api/test-cases/:id`: 删除案例
  - `POST /api/test-cases/batch`: 批量操作
  - `GET /api/test-cases/stats`: 获取统计信息
  - `GET /api/test-cases/api-names`: 获取所有接口名分组
  - `GET /api/test-cases/tags`: 获取所有标签
- 添加中间件：
  - 数据库连接初始化
  - 请求参数验证
  - String 转 JSON 处理

### 3. 注册路由 (src/http-server/static.js)
- 导入 `test-case-api.js` 路由
- 在 `/api/test-cases` 路径挂载
- 添加前端页面路由 `/test-case-manager`

### 4. 创建前端管理页面 (public/test-case-manager/)
- 创建目录 `public/test-case-manager`
- 实现单页应用 `index.html`
- 技术栈：
  - Tailwind CSS (UI 框架)
  - Alpine.js (响应式数据绑定)
  - Lucide Icons (图标库)
  - Monaco Editor (JSON 编辑器，可选)

#### 页面功能模块：
**a. 侧边栏导航**
- 系统 Logo
- 菜单项：案例列表、新增案例、统计分析
- 用户信息

**b. 顶部工具栏**
- 搜索框（接口名、标题）
- 标签过滤器（多选）
- 接口名分组筛选
- 新建案例按钮

**c. 主内容区 - 列表视图**
- 表格展示案例列表
- 列：接口名、标题、标签、请求时间、操作
- 支持展开查看详情（请求参数、返回数据）
- 分页组件

**d. 创建/编辑表单**
- 接口名称（输入框）
- 案例标题（输入框）
- 请求参数（文本域 + String 转 JSON 按钮）
- 返回数据（文本域 + String 转 JSON 按钮）
- 备注（文本域）
- 标签（标签输入组件）
- 请求时间（日期选择器）

**e. 详情查看**
- JSON 格式化展示请求参数
- JSON 格式化展示返回数据
- 支持复制功能

### 5. 实现 String 转 JSON 工具函数
- 在前端实现 `stringToJson()` 函数
- 支持智能识别 JSON 字符串
- 错误提示与格式化

### 6. 更新工具箱首页
- 在 `public/index.html` 添加测试案例管理系统入口
- 添加到 `bin/dev.js` 的常用页面列表中

## 📝 注意事项

- 使用 JSONDB 的 JSONB 模式节省存储空间
- 启用查询缓存提升性能
- JSON 编辑器使用 Monaco Editor 或 CodeMirror（可选）
- 标签使用数组存储，支持多标签
- 所有时间使用 ISO 8601 格式
