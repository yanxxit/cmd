# File Viewer 待优化方向（第四轮 - 2026-03-10）

## 概述

本文档列出了本地文件查看器 (file-viewer) 项目在已有三轮优化基础上的 3 个新增待优化方向。

## ✅ 优化方向一：文件多选操作（已完成）

### 问题描述
当前只能单个操作文件，无法批量操作：
- 无法选择多个文件进行删除
- 无法批量下载多个文件
- 无法批量重命名文件

**用户需求**:
- Ctrl/Cmd + 点击选择多个文件
- Shift + 点击选择连续范围
- 批量删除、批量下载

### 优化目标
- Ctrl/Cmd + 点击多选
- Shift + 点击连选
- 批量删除功能
- 显示已选择文件数量和列表

### 实现内容
- ✅ 添加 `state.selectedFiles` 和 `state.lastClickedIndex` 状态
- ✅ 实现 `handleFileClickWithSelect()` 函数
- ✅ 实现 `toggleFileSelection()` 切换选择
- ✅ 实现 `selectRange()` 范围选择
- ✅ 多选工具栏 UI 和样式
- ✅ 批量下载功能
- ✅ 批量删除功能（带确认）
- ✅ Esc 键清除选择

### 优先级
🔴 高（提升操作效率）

---

## ⏸️ 优化方向二：文件复制和移动（待实现）

### 问题描述
当前无法复制和移动文件：
- 无法复制文件到其他目录
- 无法移动文件到其他目录
- 无法使用快捷键 Ctrl+C/Ctrl+V

**对比参考**:
- Windows 资源管理器：Ctrl+C 复制，Ctrl+V 粘贴
- macOS Finder：Cmd+C 复制，Cmd+V 移动

### 优化目标
- Ctrl/Cmd + C 复制文件
- Ctrl/Cmd + V 粘贴到当前目录
- 右键菜单支持复制/移动
- 显示复制/移动进度

### 优先级
🟡 中（提升用户体验）

---

## ⏸️ 优化方向三：面包屑导航优化（待实现）

### 问题描述
当前面包屑导航存在不足：
- 路径过长时显示不全
- 无法快速跳转到历史路径
- 缺少路径快速复制功能

**用户体验问题**:
- 深层目录路径显示困难
- 频繁返回上级目录操作繁琐
- 无法快速获取当前路径

### 优化目标
- 路径过长时自动折叠中间部分
- 点击面包屑显示下拉历史路径
- 添加一键复制当前路径按钮

### 优先级
🟢 低（功能增强型优化）
