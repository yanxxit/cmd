# Git 日志可视化服务器 - 功能说明

## 📋 概述

Git 日志可视化服务器是一个基于 Vue 3 + Tailwind CSS 的 Web 应用，提供强大的 Git 提交历史查看、代码对比、团队效率分析和智能搜索功能。

**访问地址：** `http://127.0.0.1:3000/`

---

## 🚀 启动方式

```bash
# 启动服务器（默认端口 3000）
x-git-log-server

# 指定端口
x-git-log-server -p 3000

# 启动并自动打开浏览器
x-git-log-server --open
```

---

## 📱 页面功能

### 1️⃣ 提交列表（默认页面）

**功能特点：**
- 📅 多日期范围筛选（今天/昨天/最近 7 天/最近 30 天/自定义）
- 👤 用户筛选（从最近提交的用户中快速选择）
- 🔍 作者过滤（支持姓名或邮箱）
- ✅ 仅我的提交 / 排除 Merge 提交

**统计卡片：**
- 总提交数
- 修改文件数
- 新增行数
- 删除行数

**交互功能：**
- 点击提交卡片展开查看详情
- 点击文件查看代码差异（Diff）
- 支持复制 Diff 内容

---

### 2️⃣ 代码对比

**功能特点：**
- 🔀 对比任意两个提交之间的差异
- 📊 显示修改文件数量、总变更数统计
- 📝 支持详细模式（包含行级解析）

**使用示例：**
- 旧提交：`HEAD~1`（或任意 commit hash）
- 新提交：`HEAD`（或任意 commit hash）

---

### 3️⃣ 统计分析

#### 🔥 提交热力图
- 按小时统计提交分布（0-23 点）
- 可视化条形图展示活跃时段
- 支持自定义日期范围

#### 📅 星期分布
- 显示一周每天的提交数量
- 彩色编码区分不同日期
- 识别团队工作模式

#### 🏆 贡献者排名
- 按提交数排序的贡献者列表
- 显示排名（🥇🥈🥉）
- 包含邮箱和提交数统计

---

### 4️⃣ 智能搜索

#### 📁 按文件搜索
- 搜索特定文件的提交历史
- 支持完整路径或部分路径
- 使用 `--follow` 追踪文件重命名

**示例：**
- `src/git/index.js`
- `bin/git-log-server.js`
- `package.json`

#### 🔑 按关键词搜索
- 搜索提交消息中的关键词
- 支持日期范围过滤
- 支持作者过滤
- 忽略大小写匹配

**示例：**
- 关键词：`fix`, `feat`, `bug`, `optimize`
- 结合日期：查找某段时间的特定类型提交

## 📡 API 接口

### 提交相关

```bash
# 获取提交列表
GET /api/commits?date=yesterday&author=xxx&userId=0&noMerges=true&mine=true

# 获取提交详情
GET /api/commits/:hash

# 获取提交中的文件 diff
GET /api/commit-diff?hash=xxx&filePath=yyy
```

### 代码对比

```bash
# 多提交对比
GET /api/diff/compare?oldHash=xxx&newHash=yyy&detailed=true

# 解析 diff 内容（POST）
POST /api/diff/parse
Body: { "content": "diff 内容" }
```

### 统计分析

```bash
# 代码变更热力图
GET /api/stats/heatmap?filePath=xxx&since=2024-01-01&until=2024-12-31

# 贡献者统计
GET /api/stats/contributors?since=2024-01-01&until=2024-12-31
```

### 智能搜索

```bash
# 按文件路径搜索
GET /api/search/file?path=src/**/*.js&limit=50

# 按关键词搜索
GET /api/search/message?keyword=fix&since=2024-01-01&author=xxx&limit=50
```

### 用户相关

```bash
# 获取最近提交用户列表
GET /api/users?limit=10
```

## ⚠️ 注意事项

1. **Git 仓库要求**：必须在 Git 仓库目录下启动
2. **端口占用**：确保 3000 端口未被占用
3. **浏览器兼容**：推荐使用 Chrome/Edge/Firefox 最新版
4. **大数据量**：大量提交时建议使用日期范围筛选

## 🐛 常见问题

### Q: 页面空白无数据？
```bash
# 检查是否在 Git 仓库目录
git status

# 查看服务器日志
# 启动时添加 --verbose 参数（如有）
```

### Q: 用户筛选不生效？
```
1. 确认已加载用户列表（查看"显示用户筛选"）
2. 点击用户按钮进行筛选
3. 点击"应用筛选"刷新数据
```

### Q: Diff 内容加载失败？
```
1. 检查文件路径是否正确
2. 确认提交 hash 存在
3. 尝试刷新页面
```
