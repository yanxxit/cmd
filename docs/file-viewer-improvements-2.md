# File Viewer 待优化方向（第二轮）

## 概述

本文档列出了本地文件查看器 (file-viewer) 项目在已有优化基础上的 3 个新增待优化方向。

**检查时间**: 2026-03-10  
**实施状态**: ✅ 全部完成

---

## ✅ 优化方向一：新建文件功能（已完成）

### 问题描述
当前右键菜单中有"新建文件"选项，但点击后仅提示"新建文件功能暂未实现"，用户体验不一致。

**代码位置**: `app.js` - `createFile()` 函数

### 优化目标
- 实现新建文件功能
- 支持创建空文本文件
- 创建后自动刷新文件列表

### 技术方案
后端添加 `POST /api/file/create` 接口，前端调用该接口创建空文件。

### 实现内容
- ✅ 后端 API：`POST /api/file/create` 创建空文件
- ✅ 前端调用：`createFile()` 函数实现
- ✅ Toast 提示：成功/失败提示
- ✅ 缓存清除：创建后自动刷新列表

### 优先级
🔴 高（功能不一致，影响用户体验）

---

## ✅ 优化方向二：Toast 提示替代 alert（已完成）

### 问题描述
当前错误提示使用浏览器原生 `alert()` 对话框，存在以下问题：
- 阻塞页面，用户体验差
- 样式与整体 UI 不统一
- 无法自定义显示位置和时间

**代码位置**: `app.js` 中共有 8 处 `alert()` 调用

### 优化目标
- 实现非阻塞式 Toast 提示
- 样式与整体 UI 统一
- 支持成功/错误/警告等不同类型
- 自动消失，无需用户操作

### 技术方案
创建 Toast 组件，提供 `showToast(message, type, duration)` 方法。

### 实现内容
- ✅ Toast 容器：固定在页面右上角
- ✅ 四种类型：success、error、warning、info
- ✅ 动画效果：滑入/滑出动画
- ✅ 自动消失：3 秒后自动隐藏
- ✅ 替换所有 alert：8 处全部替换

### 优先级
🟡 中（影响用户体验，但不影响功能）

---

## ✅ 优化方向三：搜索功能增强（已完成）

### 问题描述
当前搜索功能存在以下不足：
- 文件名搜索不支持模糊匹配高亮
- 代码搜索结果中匹配内容未高亮显示
- 搜索结果无法快速定位到文件位置

**代码位置**: `app.js` - `performSearch()` 和 `createSearchResultItem()`

### 优化目标
- 文件名搜索结果高亮关键词
- 代码搜索结果中高亮匹配内容
- 支持点击搜索结果跳转到文件位置

### 技术方案
- 使用正则表达式高亮匹配文本
- 代码搜索结果中添加"打开文件"按钮
- 点击后在模态框中打开文件并定位到匹配行

### 实现内容
- ✅ 文件名高亮：`highlightText()` 函数
- ✅ 代码高亮：`highlightCodeMatches()` 函数
- ✅ 搜索样式：`.search-highlight` 高亮样式
- ✅ 深色模式：适配深色主题的高亮颜色

### 优先级
🟢 低（功能增强型优化）
